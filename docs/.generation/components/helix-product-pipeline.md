# helix-product-pipeline documentation impact rules

Update public docs when changes affect:

- Product rendering inputs, output formats, HTML structure, JSON responses, JSON-LD, sitemaps, merchant feeds, or indexes.
- URL pattern handling, fallback behavior, authored content overlay behavior, or cache keys visible to site implementers.
- Template conventions, configuration fields, examples, or troubleshooting guidance.

Do not update public docs for internal renderer refactors, worker implementation details, or performance-only changes unless they alter documented behavior or operational guidance.
