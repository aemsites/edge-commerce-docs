Payments

Initiate a payment

POST /{org}/sites/{site}/orders/{orderId}/payments

Base URL https://api.adobecommerce.live · Staging https://api-stage.adobecommerce.live

Validates the request, looks up the payment provider, enforces the pending order-state guard (with idempotent replay), and delegates to the provider. Returns the provider result (synchronous charge outcome or a requires_action redirect). No authentication required.

Path parameters

NameTypeRequiredDescription
orgstringyesOrganization identifier.
sitestringyesSite identifier.
orderIdstringyesOrder identifier (full or friendly id).

Request headers

NameTypeRequiredDescription
AuthorizationstringnoBearer token. `Bearer <jwt>`. Optional — guest checkout is supported.
Content-TypestringyesMust be `application/json`.

Request body

FieldTypeRequiredDescription
providerstringyesRegistered provider name (e.g. chase, chase-wallet, paypal, paypal-express, affirm).
paymentMethodstringyesPayment method (e.g. card, wallet).
idempotencyKeystringyesCaller-supplied idempotency key; replays return the prior result.
fraudTokenstringnoOptional fraud-provider JS SDK session token.
