Customers

Retrieve an address or order

GET /{org}/sites/{site}/customers/{email}/{subroute}/{resourceId}

Retrieves a single address (requires ownership of the email) or a single order. Order lookup is intentionally unauthenticated — knowing email + orderId is treated as sufficient proof for guest order-status lookups.

Path parameters

NameTypeRequiredDescription
orgstringyesOrganization identifier.
sitestringyesSite identifier.
emailstringyesCustomer email address.
subroute"addresses" | "orders"yesEither `addresses` or `orders`.
resourceIdstringyesAddress id or order id.

Request headers

NameTypeRequiredDescription
AuthorizationstringyesBearer token. `Bearer <jwt>`.

Response

200 The address or order document.

404 No such address or order for this customer.
